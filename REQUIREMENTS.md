## 1. Core Objectives

PathFinder is an autonomous AI job discovery and matching system designed for TPM (Technical Program Manager) job seekers. The system coordinates four independent Agents to automatically discover AI companies, scrape TPM job postings, match resumes to job descriptions, and optimize resume tailoring, with results persisted to a local Excel dashboard. The development process is assisted by 8 Claude Code Custom Agents (planning + coordination + quality analysis) and 8 Skills (coordination + operational execution).

## 9. PRJ-002: 3-Dimension Scoring Requirements

**Project**: PRJ-002 — 3-Dimension Scoring (ATS / Recruiter / HM). See `docs/sdlc/PRJ-002-3d-scoring/` for BRD + tech design.

**Motivation**: The single LLM-derived "fit score" did not map to the real North American hiring funnel (ATS keyword filter → recruiter quick scan → hiring manager deep dive). Score Delta after tailoring conflated keyword gains with semantic strength changes, leaving the user no signal on which one moved.

## 10. Technical Decision Record

### DEC-001: LLM Model Selection — Retain Gemini flash-lite (2026-03-16)

**Background**: Evaluated whether to switch to a stronger model (GPT-5.1, GPT-5.1-mini, Claude Sonnet/Opus) to improve match scoring quality.

**Workload**: ~50-100 companies, 100-200 jobs, full run approximately 500-1000 API calls.

**Free tier comparison**:

| Model | RPM | TPM | RPD | Can it handle full workload |
|------|-----|-----|-----|-------------|
| **Gemini 3.1 flash-lite** | 15 | 250k | 500 | Yes (~33 minutes) |
| GPT-5.1 | 3 | 10k | 200 | No — RPD insufficient, TPM too low (3 JDs hit the limit) |
| GPT-5.1-mini | 3 | 60k | 200 | No — RPD insufficient |
| Claude Sonnet/Opus | 5 | 10k | 200 | No — RPD insufficient, TPM too low |

**Decision**: Retain Gemini 3.1 flash-lite as the sole model.

**Rationale**:
1. Other free models' RPD (200) cannot cover the 500-1000 calls needed for a full run
2. GPT-5.1 and Claude's TPM (10k) is too constrained for long-text tasks like JD extraction/matching
3. Current matching quality meets requirements; no need to introduce multi-model complexity for quality improvement
4. If a stronger model is needed in the future, a hybrid strategy can be adopted: Gemini coarse screening for full volume -> strong model fine evaluation for top 20-30 (within RPD limits)
